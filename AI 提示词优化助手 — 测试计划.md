**文档版本**：V1\.0

**技术栈**：Flask 2\.x、SQLite、OpenAI 兼容 API（DeepSeek）、原生 HTML/JS 前端

## 1 测试范围与目标

### 1\.1 功能范围

|模块|覆盖功能点|
|---|---|
|用户认证|注册、登录、登出、忘记密码、/api/me 会话恢复|
|提示词优化|多模式（通用 / 写作 / 代码 / 数据）、同步 /api/polish、流式 SSE /api/polish/stream、文本长度校验|
|继续优化|refine 接口、优化方向合并、思考过程 / 直接答案拦截与重试|
|历史记录|分页查询、删除记录、仅本人可见|
|评分|1–5 分评价、覆盖更新、权限校验|
|前端交互|微信式对话、Markdown 渲染、对比视图、收藏（localStorage）、继续优化弹窗、字体调节|
|数据与运维|SQLite 持久化、启动备份、failed 状态记录、登录限流|

### 1\.2 非功能范围

|类型|关注点|
|---|---|
|性能|优化接口响应时间、SSE 首包时间、历史分页、并发登录 / 优化（Locust）|
|安全|未登录访问拦截、越权删除 / 评分、登录暴力破解限流、Session、敏感配置不落库|
|兼容性|Chrome / Edge / Firefox 最新两个大版本；分辨率 1280×720 及以上|
|可靠性|AI 超时 / 异常时失败记录、数据库路径固定、重启后数据不丢|
|易用性|操作流程、错误提示（Toast）、空状态（收藏 / 历史）|

### 1\.3 不在范围

1. 第三方 AI 模型内容质量的主观评审（仅测接口契约与异常处理）

2. 生产级邮件验证码找回密码（当前为实训简化版）

3. 收藏云端同步（当前仅浏览器 localStorage）

4. 多租户、水平扩展与 K8s 部署（假设：单机部署）

### 1\.4 核心测试目标

1. 验证核心业务路径（注册 → 登录 → 优化 → 历史 / 评分 → 继续优化）可稳定完成。

2. 接口与前端行为符合 API 约定，异常场景有明确错误码与提示。

3. 自动化回归覆盖率满足 CI 门禁（单元 \+ API ≥ 90% 用例通过）。

4. 识别 P0/P1 缺陷并在发布前清零或制定规避方案。

## 4 测试用例设计方法

### 4\.1 采用的设计技术

|技术|应用场景|
|---|---|
|等价类划分|注册（合法 / 重复用户名 / 空密码）、评分（1–5 / 非法值）|
|边界值分析|原文 2000 字、继续优化 polished 3500 字、direction 800 字、分页 per\_page 上限 50|
|场景法|登录 → 流式优化 → 对比 → 收藏 → 继续优化 → 历史删除|
|错误推测|AI 超时、返回调研答案、输出思考过程、未登录调 API|
|状态迁移|polish\_record\.status：success / failed|

### 4\.2 关键模块用例示例

#### 模块 A：提示词优化（POST /api/polish/stream）

|用例 ID|标题|前置条件|步骤|预期结果|
|---|---|---|---|---|
|POL\-01|合法文本流式优化|已登录|POST text≤2000，mode=code|200，SSE 有 chunk，最终 record\_id，status=success|
|POL\-02|空文本|已登录|text=\&\#34;\&\#34;|400，「文本不能为空」|
|POL\-03|超长文本|已登录|text 长度 2001|400，「文本过长」|
|POL\-04|未登录|无 Session|任意合法 text|401|
|POL\-05|AI 超时|Mock 抛 API\_TIMEOUT|合法 text|503，DB 记录 failed|

#### 模块 B：继续优化（refine: true）

|用例 ID|标题|步骤|预期结果|
|---|---|---|---|
|REF\-01|正常继续优化|polished\+direction 合法|200，返回新提示词正文，历史 original 含「继续优化」|
|REF\-02|空优化方向|direction=\&\#34;\&\#34;|400|
|REF\-03|思考过程拦截|模拟模型返回「首先我需要…」|重试后成功或 400 明确错误提示|

#### 模块 C：登录限流

|用例 ID|标题|步骤|预期结果|
|---|---|---|---|
|AUTH\-RL\-01|连续错误登录|错误密码 \&gt; LOGIN\_RATE\_LIMIT 次 / 分钟|429 或业务定义限流响应|

#### 模块 D：前端收藏（手工）

|用例 ID|标题|预期结果|
|---|---|---|
|FAV\-01|收藏后进入「我的收藏」|列表展示原文 / 优化文，可复制、继续优化|
|FAV\-02|取消收藏|列表移除，气泡按钮恢复「收藏」|

## 5 测试数据

### 5\.1 数据准备方式

|数据类型|准备方式|
|---|---|
|用户账号|pytest fixture 动态创建（testuser / 随机前缀）；性能测试 perfuser\_\*|
|优化记录|API 调用或 db\_session 直接插入 PolishRecord|
|AI 响应|MOCK\_AI=1 或 conftest\.mock\_openai 固定 Mock 文本|
|继续优化|预制 polished 长文本 \+ 短 direction 样本集|

### 5\.2 数据隔离策略

1. 自动化：测试库使用 `sqlite:///:memory:` 或独立测试文件，每用例 / 模块隔离。

2. 手工：使用专用测试账号，不与演示 / 生产账号混用。

3. 性能：Locust 虚拟用户独立注册，避免污染主库（建议：性能环境独立 DB 文件）。

### 5\.3 敏感数据与脱敏

|数据|处理|
|---|---|
|DEEPSEEK\_API\_KEY|仅存 \.env，不入库、不入 Git；测试报告打码 sk\-\\*\\*\\*\\*|
|用户密码|测试密码统一为 password123 等弱口令，仅限测试环境|
|优化内容|报告摘录时截断至 200 字；不含真实个人身份信息|

## 6 缺陷管理

### 6\.1 严重级别（Severity）

|级别|定义|示例|
|---|---|---|
|S1 致命|系统不可用或核心数据丢失|无法登录、优化后 DB 未写入且前端显示成功|
|S2 严重|核心功能不可用无替代|流式优化中断、继续优化始终返回思考过程|
|S3 一般|功能受损有绕行|收藏旧数据迁移提示不友好、对比视图样式错乱|
|S4 轻微|体验 / 文案 / 偶发|Toast 文案、字号按钮对齐|

### 6\.2 优先级（Priority）

|优先级|说明|
|---|---|
|P0|发布阻塞，24h 内修复|
|P1|当前迭代必须修复|
|P2|可延后至下一迭代|
|P3|建议改进|

### 6\.3 提交规范

每条缺陷至少包含：**标题、环境、复现步骤、实际 / 预期结果、截图 / 日志、关联用例 ID、Severity、Priority、发现版本**。

### 6\.4 跟踪流程

新建 → 确认 → 指派开发 → 修复 → 待验证 → 关闭
                ↓
              延期 / 拒绝（需说明原因）

工具：实训可用 GitHub Issues；企业环境可替换为 Jira / 禅道。

## 8 风险与应对

|风险|影响|概率|缓解措施|
|---|---|---|---|
|DeepSeek API 不稳定 / 限额|联调阻塞|中|默认 MOCK\_AI=1；Staging 设配额与重试|
|AI 输出不可控（思考过程 / 直接答题）|继续优化体验差|中|启发式检测 \+ 重试；人工抽样；提示词持续调优|
|Selenium 环境依赖 Chrome|E2E 失败|中|CI 安装 Chrome；本地提供 headless 说明|
|SQLite 多环境路径混淆|数据「丢失」误判|低|固定 DB\_FILE；测试独立库|
|文本长度限制导致 refine 失败|用例失败|中|前后端分拆传参；边界用例覆盖|
|实训周期紧|覆盖不足|高|优先 P0 路径；自动化优先 API|

## 9 验收标准

### 9\.1 量化指标

|指标|通过标准|
|---|---|
|用例通过率|计划内用例通过率 ≥ 95%（不含标记 N/A 的项）|
|自动化通过率|pytest tests/unit tests/api 100% 通过方可合并 main|
|代码覆盖率|后端 backend/ 行覆盖率 ≥ 70%（实训可酌情 60%）|
|严重缺陷|S1 = 0，S2 = 0（或已评审接受并文档化）|
|P0/P1 缺陷|发布前全部关闭|
|性能（Mock）|/api/polish P95 响应 \&lt; 3s；历史列表 P95 \&lt; 500ms|
|性能（真实 API）|流式首 chunk \&lt; 10s（依赖上游，不阻断 Mock 门禁）|

### 9\.2 业务验收清单（抽样）

* [ ] 新用户可注册、登录并刷新页面保持登录态

* [ ] 四种优化模式均可完成一次流式优化

* [ ] 继续优化弹窗生成新提示词（非思考过程、非直接调研答案）

* [ ] 历史可分页、删除、评分；仅能操作本人记录

* [ ] 收藏可在「我的收藏」查看并继续优化

* [ ] AI 失败时有明确提示且历史可见 failed 记录
