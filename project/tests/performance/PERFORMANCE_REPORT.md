# 性能测试报告（Locust）

| 项目 | 内容 |
|------|------|
| 测试日期 | 2026-06-11 |
| 工具 | Locust 2.44.0 |
| 环境 | Windows 本地，MOCK_AI=1 |
| 目标地址 | http://127.0.0.1:5000 |
| 并发用户 | 5 |
| .spawn 速率 | 1 用户/秒 |
| 持续时间 | 45 秒 |

## 测试场景

| 场景 | 接口 | 权重 |
|------|------|------|
| 历史查询 | GET `/api/history?page=1&per_page=10` | 3 |
| 提示词优化 | POST `/api/polish` | 1 |

每个虚拟用户启动时完成注册与登录，保持 Session 后执行业务请求。

## 执行命令

```bash
cd project
MOCK_AI=1 locust -f tests/performance/locustfile.py --headless -u 5 -r 1 -t 45s --host http://127.0.0.1:5000
```

## 汇总结果

| 指标 | 数值 |
|------|------|
| 总请求数 | 146 |
| 失败数 | 0 |
| 错误率 | **0%** |
| 聚合 RPS | 3.36 req/s |
| 聚合平均响应时间 | 39 ms |
| 聚合 P50 | 15 ms |
| 聚合 P95 | 280 ms |

## 分接口结果

| 接口 | 请求数 | 失败率 | 平均(ms) | P50(ms) | P95(ms) | Max(ms) |
|------|--------|--------|----------|---------|---------|---------|
| GET /api/history | 100 | 0% | 15 | 13 | 27 | 64 |
| POST /api/polish | 36 | 0% | 34 | 33 | 43 | 52 |
| POST /api/login | 5 | 0% | 284 | 280 | 310 | 310 |
| POST /api/register | 5 | 0% | 325 | 320 | 400 | 402 |

> 注册/登录仅在 `on_start` 执行一次，响应时间含建连与首次处理开销，不代表稳态业务接口性能。

## 结论

在 5 并发用户、45 秒压测下：

1. **历史查询**与**提示词优化**接口错误率为 0%，P50 分别为 13ms、33ms，满足实训演示场景性能要求。
2. 高频查询接口 `/api/history` 占主要流量（约 68%），响应稳定。
3. 未发现明显超时或服务不可用情况。

## 建议

- 真实 DeepSeek API 环境下 `/api/polish` 延迟将显著高于 Mock，建议答辩演示使用 MOCK 或提前预热。
- 生产环境应使用 gunicorn 多 worker，并考虑数据库升级与限流策略。
